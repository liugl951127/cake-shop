4

关闭"合法域名校验" (本地调试)

工具栏 → 详情 → 本地设置 → 勾选"不校验合法域名"

  • 开发期: 勾上,允许 HTTP / 任意域名
  • 生产期: 取消勾选,必须 HTTPS + 配置域名
  • v36.4 默认 http://127.0.0.1:8080,无需此选项
⚙️ 本地设置

[ ] 启用多核心编译

自定义预处理 (Condition):
globalThis.__BACKEND_URL = 'https://api.cakeshop.com'
5

配置后端地址

3 种方式,选 1 个:

  • A. 改源码: utils/request.js DEFAULT_BACKEND_URL
  • B. app.js: 取消注释 globalThis.__BACKEND_URL = '...'
  • C. 自定义预处理 (推荐,见上一步)
🌐 后端配置
// utils/request.js
// 默认 (本地)
DEFAULT_BACKEND_URL = 'http://127.0.0.1:8080'
─────────────
// 优先级
globalThis.__BACKEND_URL || DEFAULT_BACKEND_URL
─────────────
线上: 'https://api.cakeshop.com'
本地: 'http://127.0.0.1:8080'
6

启动 Spring Boot 后端

独立终端跑后端 (确保 127.0.0.1:8080 通):

  • mvn spring-boot:run -Dspring-boot.run.profiles=dev
  • 或 java -jar target/cakeshop-backend.jar --spring.profiles.active=dev
  • 看到 Started CakeshopBackendApplication = OK

7

点击"编译"运行

工具栏 → 编译 (Ctrl+B / Cmd+B)

  • 左侧模拟器自动加载首页
  • 控制台 (F12) 看日志,无红色错误 = OK
  • v36.4 默认本地后端,可直接登录 admin/admin123

发布前配置 (生产环境)

微信公众平台配置:

  • 开发 → 开发设置 → 服务器域名 → 添加 https://api.cakeshop.com
  • request 合法域名 · uploadFile 合法域名 · downloadFile 合法域名
  • 必须 HTTPS + ICP 备案
  • 工具栏 → 上传 (Ctrl+Shift+U) → 填版本号

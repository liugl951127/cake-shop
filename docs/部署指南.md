# 详细部署指南(完整版)

> 涵盖授权登录、安全支付、订单全流程、地址增强等的部署

## 一、注册小程序

1. https://mp.weixin.qq.com 注册小程序
2. 记下 **AppID**(小程序后台 -> 开发 -> 开发管理 -> 开发设置)

## 二、安装微信开发者工具

https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

## 三、导入项目

1. 开发者工具 -> 项目 -> 导入项目
2. 项目目录: `cake-shop/miniprogram`
3. AppID: 你的 AppID
4. 后端服务: **必须选「微信云开发」**

## 四、修改 appid(关键)

`miniprogram/project.config.json`:
```json
"appid": "wx你的实际appid"
```

## 五、开通云开发 & 配置环境变量

1. 工具栏点「云开发」-> 开通 -> 建环境 -> 记下 **环境 ID**
2. 改 `miniprogram/app.js`:
   ```js
   cloudEnvId: 'cloud-xxxxx',  // 你的环境 ID
   ```

3. **配置支付环境变量**(云开发控制台 -> 云函数 -> 你的环境 -> 配置):
   ```
   MCH_ID     = 你的微信支付商户号
   MCH_KEY    = 32 位 API 密钥
   NOTIFY_URL = https://你的回调域名/payCallback (可选)
   ```
   演示模式:不配也能跑,会自动走"下单成功"流程(不真扣款)

## 六、上传云函数

**所有**以下云函数都要上传:

| 模块 | 云函数 |
|------|------|
| 登录 | login / getPhone / logout |
| 商品 | getCategories / getGoods / getGoodsDetail |
| 订单 | addOrder / getOrders / getOrderDetail / cancelOrder / confirmReceive / refund |
| 支付 | payCallback |
| 地址 | getAddress / addAddress / updateAddress / deleteAddress / setDefaultAddress / getRegions |
| 收藏 | addFavorite / removeFavorite / getFavorites |
| 管理 | adminAddGoods / adminUpdateGoods / adminDeleteGoods / adminGetOrders / adminUpdateOrder / adminShipOrder / addCategory / uploadFile |
| 定时 | autoCloseOrders / autoConfirmReceive |
| 数据 | initData / initRegions |

右键 `cloudfunctions` -> 上传并部署:云端安装依赖

## 七、初始化数据库

### 创建集合

云开发控制台 -> 数据库 -> 创建:

- users
- categories
- goods
- orders
- orderLogs   ← 新增:订单日志
- refunds     ← 新增:退款记录
- addresses
- favorites
- banners
- regions     ← 新增:省市区数据

### 灌入数据

| 云函数 | 入参 | 作用 |
|------|------|------|
| `initData` | `{}` | 灌入 5 分类 + 9 商品 + 3 轮播 |
| `initRegions` | `{}` | 灌入省市区数据(覆盖一线+新一线) |

### 配置权限

参考 `docs/数据库集合权限.md`,**orderLogs / refunds** 加为:
- 读:仅创建者
- 写:仅云函数

## 八、配置定时触发器(关键!)

云开发控制台 -> 云函数 -> `autoCloseOrders` -> 触发管理:

```
Cron 表达式: 0 */5 * * * *   // 每 5 分钟执行
```

云开发控制台 -> 云函数 -> `autoConfirmReceive` -> 触发管理:

```
Cron 表达式: 0 0 3 * * * * // 每天凌晨 3 点执行
```

## 九、设置自己为管理员

1. 开发者工具预览,触发登录
2. 云开发 -> 数据库 -> `users` 集合
3. 找到自己,改 `isAdmin = true`

## 十、配置微信支付(可选)

1. 申请微信支付商户号:https://pay.weixin.qq.com
2. 关联小程序:商户平台 -> 产品中心 -> APPID 授权管理
3. 配支付密钥:商户平台 -> API 安全 -> APIv2 密钥(32 位)
4. 把 MCH_ID / MCH_KEY 配到云函数环境变量

## 十一、提交审核

1. 开发者工具 -> 上传代码
2. 微信公众平台 -> 版本管理 -> 提交审核
3. 审核通过 -> 发布

## 常见问题

### Q1: 支付一直"调起失败"
- 检查商户号是否关联了小程序
- 检查 MCH_ID / MCH_KEY 是否配对
- 演示模式:不配支付也能跑(直接显示下单成功)

### Q2: 订单 30 分钟没自动关
- 确认 `autoCloseOrders` 定时器已开启
- 在云函数日志里能看到执行记录

### Q3: token 经常失效
- 默认 7 天有效,如果后端重启内存缓存会清空,会触发重新登录
- 生产环境可换 Redis

### Q4: 省市区只有一线城市数据
- 当前数据覆盖:北京、上海、广州、深圳、杭州、南京、成都、武汉、西安
- 完整 3000+ 区县数据建议从淘宝/民政部公开 csv 导入
- 或者把 `initRegions/index.js` 里的 DATA 数组替换

### Q5: 手机号拿不到
- 当前用新版 `getPhoneNumber` 组件,需 button open-type="getPhoneNumber"
- 旧版 encryptedData 解密需 session_key 缓存,生产环境建议做

## 生产环境升级建议

1. **支付回调**:`payCallback` 应配置独立的 HTTP 触发器路径
2. **数据库**:`orders` 集合按 `_openid + status` 建索引;大数据量时分表
3. **缓存**:token 缓存从内存换 Redis(支持多实例)
4. **图片**:用 CDN + WebP,减少云存储费用
5. **监控**:接 Sentry / 微信小程序实时日志
6. **订单导出**:管理员加「订单导出 Excel」功能
7. **物流查询**:对接快递鸟/快递100 查物流详情

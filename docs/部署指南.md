# 详细部署指南

> 一步步带你从 0 部署到上线,跟着操作即可。

## 一、注册小程序

1. 打开 https://mp.weixin.qq.com 点击"立即注册"
2. 选择"小程序"账号类型
3. 填写邮箱(未被微信注册过)、密码,激活邮箱
4. 填写主体信息(个人/企业),个人主体也能用
5. 登录小程序后台 -> 开发 -> 开发管理 -> 开发设置 -> 复制 **AppID(小程序ID)**

## 二、安装微信开发者工具

1. 下载: https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 扫码登录

## 三、导入项目

1. 开发者工具 -> 项目 -> 导入项目
2. 项目目录: 选择 `cake-shop/miniprogram`
3. AppID: 填写你刚注册的 AppID
4. 后端服务: 选择"微信云开发"
5. 点击"导入"

## 四、修改 appid(关键一步)

打开 `miniprogram/project.config.json`,修改:
```json
{
  "appid": "wx你的实际appid"
}
```

## 五、开通云开发

1. 开发者工具顶部工具栏 -> 点击"云开发"按钮
2. 弹出开通页面 -> 选择套餐(免费版够用)
3. 创建环境:
   - 环境名称: `prod`(自定义)
   - 环境付费: 免费
4. 创建完成后,**记下环境 ID**(形如 `cloudbase-abc123`)

## 六、配置云开发环境 ID

打开 `miniprogram/app.js`:
```js
cloudEnvId: 'cloudbase-abc123',  // ← 改成你的环境 ID
```

## 七、上传云函数

### 批量上传(推荐)

1. 右键 `cloudfunctions` 文件夹
2. 选择"上传并部署:云端安装依赖"
3. 等待所有云函数上传完成(可能要几分钟)

### 单个上传

每个云函数目录右键 -> "上传并部署:云端安装依赖"

> 必须上传的云函数: login, getCategories, getGoods, getGoodsDetail, addOrder, getOrders, payCallback, addAddress, getAddress, updateAddress, deleteAddress, setDefaultAddress, cancelOrder, confirmReceive, addFavorite, removeFavorite, getFavorites, adminAddGoods, adminUpdateGoods, adminDeleteGoods, adminGetOrders, adminUpdateOrder, addCategory, initData

## 八、初始化数据库

### 1. 创建集合

云开发控制台 -> 数据库 -> 创建以下集合:
- `users`
- `categories`
- `goods`
- `orders`
- `addresses`
- `favorites`
- `banners`

### 2. 灌入演示数据

云开发控制台 -> 云函数 -> `initData` -> 测试 -> 输入 `{}` -> 确定

返回结果示例:
```json
{
  "code": 0,
  "data": {
    "inserted": { "categories": 5, "goods": 9, "banners": 3 }
  }
}
```

### 3. 配置权限

参考 `docs/数据库集合权限.md`,关键:
- users/orders/addresses/favorites: 仅创建者读写
- categories/goods/banners: 所有用户可读,仅管理员可写

## 九、设置自己为管理员

1. 第一次运行小程序(开发者工具预览)
2. 触发登录逻辑(随便点个"我的"会触发)
3. 云开发控制台 -> 数据库 -> `users` 集合
4. 找到你的记录(看 _openid)
5. 编辑,把 `isAdmin` 改为 `true`
6. 保存
7. 重新进入小程序 -> 我的 -> 出现"商品管理"和"订单管理"

## 十、真机预览 & 提交审核

### 预览
开发者工具 -> 预览 -> 扫码 -> 手机查看效果

### 上传代码
开发者工具 -> 上传 -> 填写版本号和项目备注

### 提交审核
微信公众平台 -> 版本管理 -> 找到刚上传的版本 -> 提交审核

审核通过后,点击"发布"即可上线!

## 常见问题

### Q1: 云函数调用失败,提示 cloud has not been initialized
检查 `app.js` 中 `cloudEnvId` 是否填写正确,以及 `wx.cloud.init` 是否成功执行。

### Q2: 数据库读不到数据
检查集合权限是否为"仅创建者",如需全部用户可读需改为"所有用户可读"或自定义规则。

### Q3: 支付提示"小程序未配置支付"
两种方案:
- 演示模式: 已经能跑通(直接标记订单为已支付)
- 真实支付: 申请商户号 + 接入 `cloud.cloudPay` 模块

### Q4: 上传图片失败
- 确认 `storage` 配额是否充足(免费版 5GB)
- 确认 `uploadFile` 云函数已上传

### Q5: 地图位置/分享外链打不开
小程序后台 -> 开发 -> 开发管理 -> 开发设置 -> 配置服务器域名 / 业务域名

### Q6: tabBar 图标显示不出来
- 确认图片在 `miniprogram/images/` 目录下
- 路径以 `/` 开头(在 `app.json` 中已用 `images/tab-home.png` 这种相对路径)

## 性能建议

1. **云函数冷启动**: 首次调用有 1-2 秒延迟,生产环境建议加"预热"
2. **图片优化**: 商品图建议使用 CDN + WebP 格式
3. **数据库索引**: 大数据量时务必按 `docs/数据库集合权限.md` 建立索引
4. **分页**: 列表页已经做了分页,商品量大时调整 `pageSize`

## 升级维护

- **改商品字段**: `cloudfunctions/adminAddGoods` + 前端 `pages/admin/edit/edit`
- **改订单状态**: `cloudfunctions/adminUpdateOrder` + 前端 `order/list/list` 的 `statusMap`
- **加新页面**: 在 `app.json` 的 `pages` 数组中注册

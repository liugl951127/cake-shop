# 数据库集合与权限配置

> 云开发控制台 -> 数据库 -> 创建以下集合,并按说明配置权限

## 集合清单

| 集合名 | 说明 | 权限建议 |
|------|------|----------|
| users | 用户信息 | 仅创建者可读写 |
| categories | 商品分类 | 所有用户可读,仅管理员可写 |
| goods | 商品信息 | 所有用户可读,仅管理员可写 |
| orders | 订单 | 仅创建者可读写,管理员可读 |
| addresses | 收货地址 | 仅创建者可读写 |
| favorites | 收藏 | 仅创建者可读写 |
| banners | 轮播图 | 所有用户可读,仅管理员可写 |
| notices | 公告 | 所有用户可读,仅管理员可写 |

## 一键配置（云开发控制台 -> 数据库 -> 权限设置）

### users
- 读写:仅创建者
- 自定义安全规则:
  ```json
  { "read": "doc._openid == auth.openid", "write": "doc._openid == auth.openid" }
  ```

### categories / goods / banners
- 读:所有用户
- 写:仅管理员(自定义规则):
  ```json
  { "read": true, "write": "getUserInfo('isAdmin') == true" }
  ```

### orders / addresses / favorites
- 读写:仅创建者

## 集合索引建议

为大数据量集合建立索引,云开发控制台 -> 数据库 -> 集合 -> 索引管理:

- **goods**: `{ category: 1, status: 1 }`, `{ recommend: 1, status: 1 }`, `{ name: 1 }`
- **orders**: `{ _openid: 1, status: 1, createTime: -1 }`
- **favorites**: `{ _openid: 1, goodsId: 1 }` (唯一)
- **addresses**: `{ _openid: 1, isDefault: -1 }`

## 管理员设置

默认所有用户 isAdmin = false。
将自己设为管理员(替换为你的 openid,云开发 -> 用户管理查看):

```js
// 在云开发控制台数据库 -> users 集合,找到自己,手动改 isAdmin 为 true
// 或使用云函数临时提权
```
